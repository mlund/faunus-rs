# Tabulated 6D Rigid-Body Energy: Validation Report

## Overview

This example validates the `tabulated6d` energy term in Faunus, which replaces
explicit nonbonded pair interactions with a pre-computed 6D icosphere lookup
table generated by [Duello](https://github.com/mlund/duello). The potential of
mean force (PMF) is compared across three independent methods for the CPPM-P18
system at 5 mM NaCl.

## Methods

| Label | Description |
|-------|-------------|
| **Duello** | Boltzmann inversion of 6D table (orientation-averaged free energy at each R) |
| **Explicit MC** | Faunus MC with `!Coulomb` + `!WeeksChandlerAndersen`, spline tabulation |
| **6D table MC** | Faunus MC using `tabulated6d` energy with Boltzmann-weighted interpolation |

PMFs are extracted as w(r) = -ln(g(r)) from the z-projected RDF and shifted to
match Duello at large r (80-100 A).

## Key Parameters

- Salt: 5 mM NaCl (lambda_D = 43 A)
- Table range: rmin=38.0, rmax=200.0, dr=1.0 A
- Icosphere resolution: 0.6 (42 vertices, ~15 deg angular spacing)
- Dihedral bins: 17 (omega_step ~ 21 deg)
- MC steps: 500,000 sweeps
- Temperature: 298.15 K

## Results

![PMF comparison](pmf_comparison.png)

| Source | PMF minimum (kT) | r_min (A) |
|--------|------------------|-----------|
| Duello | -4.02 | 42.0 |
| Explicit MC | -3.93 | 41.8 |
| 6D table MC | -4.29 | 41.8 |

Agreement within ~0.3 kT at the contact minimum. At r > 50 A all three
curves overlap.

## Boltzmann-Weighted Interpolation

Linear barycentric interpolation on the icosphere overestimates repulsive
energies due to Jensen's inequality: for a convex function U,

    sum_ij w_i w_j U(v_i, v_j) >= U(interpolated direction)

At contact distances where the WCA repulsive core dominates, this bias is
significant (~2 kT with 42 vertices). The fix interpolates Boltzmann factors
instead of energies:

    U_interp = u_min - ln(sum_ij w_i w_j exp(-beta (U_ij - u_min))) / beta

This is implemented in `icotable::Table6DFlat::lookup_boltzmann()`.

## Lookup Scaling

The lookup cost is O(V) per call due to a linear scan in `find_face_bary` to
locate the nearest vertex. This is called twice per energy evaluation (once per
molecule direction). Going from 42 to 92 vertices more than doubles wall time.
